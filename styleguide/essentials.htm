Crazybee Design System

The core principles behind our design system are lightweightness, flexibility, unambiguity, and ease of use. It is meant to support our company's business goals and communicate its brand identity, being easily adaptable to any new conditions as neccessary, just like the company itself.

Grids

Any page element can be told to be a grid simply by applying .grid-container class. By default, the gutter equals 1em. Gutters can be removed throughout, or between cols and rows only by applying additional .grid-no-gap , .grid-no-row-gap , .grid-no-col-gap classes.

Elements within a grid should use .col-2 , .col-3 , .col-4 , .col-6 , .col-9 classes to specify their width, otherwise they will be given default width according to the grid type. Elements can be set to start from an arbitrary column by using .start-col-* classes, such as .start-col-2 or .start-col-12.

Misc

There is also a handful of additional general-purpose CSS classes that may be sometimes helpful to you.

.no-pad removes all padding from the element
.no-pad-b removes just the bottom padding
.extra-pad-t adds more padding to the top
.extra-marg-t adds more margin to the top
.text-center sets text alignment to centered

Look into main.css to learn about other modifiers that might have been missed in this documentation.

Conventions

Naming

All classes and identifiers should be lowercase and use hyphens as word separators. When naming new elements, try to be as concise as possible, but in the same time make sure it is clear what's the element's purpose. Related names should be clustered, starting with the same prefix. For example:

.nav-bar , .nav-logo

For general styling you should stick to the BEM (Block-Element-Modifier) methodology. More about it can be read here: BEM 101.
